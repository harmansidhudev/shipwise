# Validate Idea

> Phase 1: DESIGN | Sprint 5 (planned)

Guides founders through rigorous idea validation before writing code.

## Coverage

- Problem definition framework (single-sentence format with severity scoring)
- Persona template (demographics, JTBD, pain points, budget, channels)
- User interview script (20 questions focused on current workflow, not feature validation)
- Competitive analysis framework (direct/indirect, feature gaps, pricing, review sentiment)
- TAM/SAM/SOM bottom-up calculator with worked example
- Landing page test design (CTA, traffic, conversion thresholds)
- Concierge/Wizard-of-Oz MVP methodology
- Go/no-go success metrics (define before experimenting)

## Checklist Items

### Problem Definition
<!-- beginner -->
**Define the problem you're solving** — Write a single sentence: "[Target user] struggles with [problem] because [reason], which costs them [consequence]." Score severity 1-10. If below 7, reconsider.

<!-- intermediate -->
**Problem statement** — Single sentence with severity score. Below 7 = weak signal.

<!-- senior -->
**Problem statement** — Severity-scored, single sentence format.

### User Interviews
<!-- beginner -->
**Talk to 5+ potential users** — Before building anything, talk to real people who have the problem. Ask about their CURRENT workflow (not whether they'd use your thing). The Mom Test: ask about their life, not your idea.

<!-- intermediate -->
**User interviews (5+ minimum)** — Focus on current workflow and pain points. Avoid leading questions about your solution.

<!-- senior -->
**User interviews** — 5+ conversations, current-workflow focused.
